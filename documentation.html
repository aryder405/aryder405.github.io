Title: Moof.py class diagram

Description: This is a class diagram of the moof.py script. It also includes data.py, which is much smaller.

Reflection: This was my first attempt at modeling a class. The class diagram can be helpful when trying to understand how it works. I made this because at the time, we needed to understand the script at a higher level. Instead of delving into all the functional details of the script that might be hard to understand, I started at a higher level and just mapped out the classes, the variables, and functions. It gave us a better understanding of the script, which then helped us understand all the functional details at a future time.

Title: Ts.go data flow diagram

Description: This is a data flow diagram of the functional design of the ts.go parser. This was delivered with the ts.go code as functional documentation.

Reflection: I made this diagram with the help of Chris, and used Visio as the software to create it. I enjoyed using Visio and have used it ever since to create any diagrams. The data flow diagram gives a much more in depth representation of how this mp2 parser program works. It took a while to create because of the complexities of the ts parser program, and forced us to delve deeper into the program which in turn made us more knowledgeable of how it worked.
