Title: Tentative Schedule

Description: This is a schedule that I created during the beginning of the project. The purpose of this schedule was to make people aware of the time scale of our project and how we could best plan out our iterations and presentations. The making of this schedule actually showed us how we were following our spiral model incorrectly and lead us into re-thinking our model and ultimately adopting the win-win spiral model.

Reflection: The reason I’m including this artifact is not because it was a good schedule that we followed and adhered to. It is because the creation of this schedule actually made us realize that we were not following the spiral model correctly. When I created this schedule, we were planning out how we were going to finish this product and therefore we planned out an iteration for each 3 major objective that we had. This way of thinking is not correct when following an agile process model because you have no idea of when each iteration will start or end. Dr. Paul noticed this immediately and enlightened us that we were not following an agile process model by planning out three iterations based upon our three major objectives. This is more of a waterfall approach to plan out an entire project. For me personally, this is what gave me greatest insight to agile versus waterfall type models. This led our team to adopt the win-win spiral model and I am very glad that we did because we have really enjoyed it and have all had the pleasure of experiencing how well the win-win spiral model can handle the creation of software models.

Title: Win win spiral model Template

Description: This is a representation of the Win Win Spiral model, with annotation boxes for each phase that can be used to describe what your team did to complete that phase.

Reflection: I was really looking for some way to keep track of each phase that our team completed. At first we used a simple bulleted format to describe each phase and how we completed them. This worked for a while but I felt there was a better way we could show the model itself while also showing our steps for each phase. So I drew a win win spiral model up in Visio and added annotation boxes for each phase. Everyone really liked it, so we adopted this as our main way to document our iterations. The only issue with this is that you have to have Visio to edit the annotation boxes.
